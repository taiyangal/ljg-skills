# ljg-plain: 白

让人 grok。靠两样东西：一个好问题，一个好类比。好问题把注意力钉在概念最卡的地方。好类比用读者已经懂的结构回答它。类比够不到的地方，就是概念自己的形状。

Grok = 好问题 + 类比 + 裂缝 - 一切多余的词

## 约束

### Org-mode 语法

- 加粗用 `*bold*`（单星号），禁止 `**bold**`
- 标题层级从 `*` 开始，不跳级

### ASCII Art

所有图表用纯 ASCII 字符。允许：`+ - | / \ > < v ^ * = ~ . : # [ ] ( ) _ , ; ! ' "` 和空格。禁止 Unicode 绘图符号。

### Denote 文件规范

- 时间戳：`date +%Y%m%dT%H%M%S`
- 可读时间：`date "+%Y-%m-%d %a %H:%M"`
- 文件名：`{时间戳}--plain-{简短标题}__plain.org`
- 输出目录：`~/Documents/notes/`

### Org 文件头

```
#+title:      plain-{简短标题}
#+date:       [{YYYY-MM-DD Day HH:MM}]
#+filetags:   :plain:atom:
#+identifier: {YYYYMMDDTHHMMSS}
#+source:     {URL 或来源描述}
```

文件写入后报告路径。

## 执行

### 1. 获取内容

URL → WebFetch | 文本 → 直接用 | 文件路径 → Read | 概念 → 直接解释 | 书名/论文名 → WebSearch

### 2. 找好问题

每个难懂的概念背后有一个卡点——读者在这里卡住，过了就通。找到它，变成问题。

方法：把自己放在读者位置，「第一次听到这个，我在哪里皱眉？」皱眉处就是卡点。

检验：回答了它，整个概念是不是就通了？

多概念输入：先看它们是不是表面长得像。如果是，好问题就是能把它们分开的那个问题。

### 3. 找类比

抓住概念的动词结构——它在干什么，什么作用于什么，什么流向什么。在日常经验里找结构一样的东西。

好类比承重：去掉它整篇文章塌掉。好类比多层：往下挖一层还像，最好三层以上。好类比自明：读者不需要解释就能看懂类比本身。

### 4. 展开

让类比自己说话。概念的每个子部分对应类比的一个部分，读者在熟悉的结构里走，新概念自然长出来。

动词优先。「分布式共识机制」→「一群人怎么在没有老大的情况下达成一致」。零术语。

遇到类比覆盖不到的子概念，补一个小类比填上，用完就走。

动词延伸检查：类比的核心动词推到新对象时，逐个测试动宾搭配——脱离类比语境，这个搭配在中文里自然吗？英文一个隐喻动词可以覆盖多个 domain（color your perception / color your memory / color your judgment），中文每个对象有自己天然的动词。不自然→换成中文对该对象天然的动词，或换句式绕开。

### 5. 找裂缝

类比做完了它能做的。现在问：概念在哪里是类比够不到的？

这不是「类比哪里错了」，是「概念还有什么」。读者的感受是「原来还有这一层」。

裂缝在类比做完全部解释之后才出现。先把类比的活干完，再超越它。

裂缝的逻辑链：首句点明类比在哪里断裂（类比能做 X，现实不能），后续每句从前句生长。检验：盖住任意一句，能从上下文推断它在说什么。做不到→有逻辑跳跃，补上过渡。「灯可以换」后面必须接「但你不知道灯有颜色」，而不能跳到另一个维度。

### 6. 留一个画面（可选）

如果文章在该停的地方自然停住了，不需要画面。硬造画面比没有画面更糟。

如果概念确实适合压成一个闭眼能看到的场景，留一个。读完全文只记住这一段，也够了。

### 7. 缝成一篇文章

步骤 2-6 是思考。这一步是写作。输出是一篇从第一行流到最后一行的连贯文章。

禁止：
- `* 好问题` / `* 类比` / `* 裂缝` 等结构标签
- 指向类比本身的元评论（「这个比喻撑不住的地方是……」）
- `**` 子标题。全文只有文件标题，正文无标题

好问题可以是开头，直接把读者扔进「卡」的状态。类比不要介绍，直接落地——不说「打个比方」，读者还没反应过来已经在类比里了。展开是类比自己在走，每推进一步概念清楚一步。裂缝从类比里长出来，不回头拆类比。画面是全文最后自然停下来的定格。

多概念：好问题在前把概念分开，类比在后分别展开。

类比做完全部工作之后裂缝才出现。先后关系，不交织。

骨架图（可选）：概念涉及空间关系时，行文中嵌入 ASCII 图（`#+begin_example` 块）。图是文章的一部分，不是附录。不需要图时不加。

### 8. 过滤

写完初稿，扫一遍：

1. 删填充——开场白、拐杖词、「值得注意的是」
2. 破公式——否定式排比全文不超过两处，三段式改两项或四项
3. 变节奏——长短句交替，段落结尾多样
4. 信任读者——跳过软化、辩解、手把手引导
5. 杀金句——听起来像可引用的，重写
6. 查译感——类比动词延伸到新对象时，动宾搭配是否中文天然？「染你记住的东西」← "color your memory" 直译，中文不这么说→换动词或换句式
7. 查跳跃——裂缝段落首句到末句，每步逻辑是否可追？前句说 A 维度，后句跳到 B 维度→补上 A→B 的桥

扫完列修改清单（哪句触发了什么，改前→改后），确认后进入下一步。清单不写入文件。

### 9. 生成 Org 文件

按 Denote 规范获取时间戳，读 `references/template.org` 确认格式，写出文件头 + 正文，存入 `~/Documents/notes/`。

## 质量标准

- 问题准：回答了它概念就通
- 类比承重：去掉类比文章塌掉
- 映射深：展开全程在类比里走
- 裂缝真：概念独有的形状，出现在类比完工之后
- 画面留：最后一行读完脑子里定格一幅图
- 零术语，费曼测试：聪明的 12 岁孩子能复述
