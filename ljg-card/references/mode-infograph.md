# 模具：信息图（-i）

## 视觉哲学

实验室精密手册感 + 波普实验风格。拒绝平庸手账，追求数据可视化的极致美感。

核心原则：**内容的结构决定视觉形式**。没有预设模块——先看内容是什么形状，再设计最贴合的视觉。

## 步骤 1：读取模板

Read `~/.claude/skills/ljg-card/assets/infograph_template.html`

模板只提供 shell（header、footer、色彩变量、网格背景、工具类）。内容区域是空白画布，所有视觉由你根据内容定制。

## 步骤 2：内容分析

- 如为 URL/文件：获取内容，提炼核心
- 如为文本：直接分析
- 目标：**识别信息的内在结构**，而非提取填充物

### 2.1 提取元信息

- **标题**：从内容提炼（≤ 15 字）
- **副标题**：一句话概括核心（≤ 30 字）
- **来源**：作者或出处（用户提供则用，否则 `李继刚`）

### 2.2 结构识别

**先问：这段信息的形状是什么？**

| 结构类型 | 识别信号 | 适合的视觉形式 |
|----------|---------|---------------|
| 循环/飞轮 | A→B→C→A，正反馈 | 环形图、螺旋、闭合箭头链 |
| 对比/对照 | 旧vs新、A vs B | 左右分屏、before/after、双轨 |
| 层级/金字塔 | 底层支撑上层 | 三角、倒金字塔、同心圆 |
| 流程/序列 | 先后顺序、步骤 | 时间轴、管道流、泳道图 |
| 分布/比例 | 数值对比、占比 | 条形图、面积块、刻度尺 |
| 矩阵/分类 | 两个维度交叉 | 2×2 象限、网格、坐标系 |
| 辐射/因果 | 一个核心多个衍生 | 星形放射、树枝、hub-spoke |
| 堆叠/积累 | 层层叠加 | 地层图、堆叠块、千层饼 |

**多数内容是复合结构**：主结构 + 辅助结构。例如"两个飞轮的对比"= 对比（主）+ 循环（子）。

识别后输出：

```
主结构：[类型] — [一句话描述]
子结构：[类型] — [一句话描述]（如有）
视觉方案：[一句话描述你打算怎么画]
```

## 步骤 3：视觉设计

### 3.1 色彩系统（固定）

不需选色。模板内置固定色盘：

| 角色 | 色值 | 用途 |
|------|------|------|
| 蓝图底 | `#F2F2F2` | 背景 + 网格纹理 |
| 鼠尾草绿 | `#B8D8BE` | 主功能区块、稳定区域、正面 |
| 荧光粉 | `#E91E63` | 关键警告、最重要单点、胜出项 |
| 柠檬黄 | `#FFF200` | 荧光笔高亮关键词（半透明） |
| 炭褐 | `#2D2926` | 线条、文字、网格、坐标 |

### 3.2 布局设计

根据识别出的结构，设计整体布局。画布宽 1080px，高度自适应。

**设计原则**：
- 主结构占据视觉重心（面积 ≥ 60%）
- 辅助信息围绕主结构排布，不喧宾夺主
- 用 CSS Grid 或 Flexbox 实现非对称布局
- 允许全宽区块、非对称分栏、大面积留白
- 循环结构用 CSS 画圆弧/箭头，不用图片

### 3.3 内容密度要求

- 关键数值用 `<strong>` 或 `.big-num` 突出
- 关键词用 `<mark>` 实现荧光笔高亮
- 使用视觉符号增加精密感（Σ、Δ、∞、→、×）
- 坐标编号可选：若内容适合实验室编目感则用，不适合则不强加

**禁止**：
- 不要生成「关键词/标签」类填充区块（信息密度极低）
- 不要为了凑数量而添加模块——3 个高质量区块好过 7 个稀薄区块

## 步骤 4：生成 HTML

将自定义 CSS 写入 `{{CUSTOM_CSS}}`，将内容 HTML 写入 `{{CONTENT_HTML}}`。

替换模板变量：

| 变量 | 内容 |
|------|------|
| `{{TITLE}}` | 主标题 |
| `{{SUBTITLE}}` | 副标题 |
| `{{CUSTOM_CSS}}` | 根据内容定制的所有 CSS |
| `{{CONTENT_HTML}}` | 根据结构设计的所有内容 HTML |
| `{{SOURCE}}` | 来源/署名 |
| `{{ARXIV_LINE}}` | 内容来自 arxiv 论文时：`<span class="arxiv">arxiv: XXXX.XXXXX</span>`；否则空字符串 |

写入：`/tmp/ljg_cast_infograph_{name}.html`

## 步骤 5：质量自检

生成 HTML 后逐项确认：

- [ ] **结构匹配**：视觉形式是否贴合内容的内在结构？（飞轮用环形，对比用分屏……）
- [ ] **色彩过滤**：是否只用了绿、粉、黄、褐？严禁五颜六色
- [ ] **主次分明**：主结构是否占据视觉重心？
- [ ] **数据密度**：关键位置是否有具体数值/品牌名？
- [ ] **taste.md 合规**：参照品味准则逐项过
